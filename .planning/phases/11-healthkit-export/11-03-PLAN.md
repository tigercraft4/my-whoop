<objective>
Integrate HealthKitExporter into the app: inject it from AppRoot, trigger authorization lazily in TodayView's `.task` modifier (only when `metrics.today != nil`), call `export()` after authorization succeeds, and show a one-time subtle "Health not connected" banner when authorization is denied. App degrades gracefully if HealthKit is unavailable (simulator, denied, restricted).
</objective>

<threat_model>
- LOW: Deep-link `x-apple-health://` may not work on all iOS versions or simulator (mitigated: fallback to `UIApplication.openSettingsURLString` if health deep link fails to open).
- LOW: `hk.authDeniedShown` UserDefaults flag persists across app reinstalls on device (until data protection resets). This is acceptable — user dismissed once is a valid signal.
</threat_model>

         - `var isAuthorized: Bool` (Published)

  <task id="11-03-T2">
    <title>Add lazy auth + export trigger in TodayView.task and "Health not connected" banner</title>
    <type>execute</type>
    <read_first>
      - ios/OpenWhoop/Tabs/TodayView.swift (full file — understand current .task modifier, existing banner patterns if any, metrics.today guard)
      - ios/OpenWhoop/Design/ (check for InformationBanner, StatusBanner, or similar reusable components; if none, use a simple SwiftUI overlay)
    </read_first>
    <action>
      Edit ios/OpenWhoop/Tabs/TodayView.swift to:

      1. Add `@EnvironmentObject private var hkExporter: HealthKitExporterViewModel` (or whatever type was created in T1).

      2. Extend the existing `.task { await metrics.refresh() }` to also trigger HealthKit auth and export:
         ```swift
         .task {
             await metrics.refresh()
             guard metrics.today != nil else { return }
             await hkExporter.requestAuthorizationAndExport(...)
         }
         ```
         OR if `.task` already chains multiple calls, add the HK call after refresh completes.

      3. Add a `@State private var showHealthBanner = false` and a `@AppStorage("hk.authDeniedShown") private var authDeniedShown = false` (or read from UserDefaults).

      4. After auth completes: if denied AND `!authDeniedShown`:
         - Set `showHealthBanner = true`
         - Set `authDeniedShown = true` (never show again)

      5. Add a subtle banner overlay inside TodayView (non-blocking, dismissible):
         - Text: "Health not connected"
         - Tap action: open `URL(string: "x-apple-health://")` via `UIApplication.shared.open`; if that fails, open `URL(string: UIApplication.openSettingsURLString)`
         - Dismiss button (X) that sets `showHealthBanner = false`
         - Banner style: small pill at top of content (not modal, not alert) — match existing WH design system colors (`WH.Color.textSecondary` background, `WH.Color.text` foreground)
         - Show only when `showHealthBanner == true`
    </action>
    <acceptance_criteria>
      - TodayView.swift contains `guard metrics.today != nil else { return }` before the HK auth call
      - TodayView.swift uses `@AppStorage("hk.authDeniedShown")` or `UserDefaults.standard.bool(forKey: "hk.authDeniedShown")` to gate the one-time banner
      - Banner is rendered conditionally (only when denied + not previously shown)
      - Banner tap attempts `x-apple-health://` deep link
      - No modal alert or blocking UI for HealthKit denial
      - App compiles and runs in simulator without crash (simulator: HKHealthStore.isHealthDataAvailable() returns false; exporter must no-op gracefully)
    </acceptance_criteria>
  </task>

  <task id="11-03-T3">
    <title>Guard HealthKitExporter against simulator / unavailable HealthData</title>
    <type>execute</type>
    <read_first>
      - ios/OpenWhoop/HealthKit/HealthKitExporter.swift (from Plan 11-02 — check if HKHealthStore.isHealthDataAvailable() guard already exists)
      - ios/OpenWhoop/App/AppRoot.swift (understand where HealthKitExporterViewModel is initialized)
    </read_first>
    <action>
      In HealthKitExporter.swift (or HealthKitExporterViewModel), add a guard at the top of `requestAuthorizationAndExport`:

      ```swift
      guard HKHealthStore.isHealthDataAvailable() else {
          // Simulator or restricted device — no-op silently
          return
      }
      ```

      This prevents crashes in the iOS Simulator where HealthKit is unavailable. The guard is the canonical Apple pattern for HealthKit apps targeting both simulator and device.
    </action>
    <acceptance_criteria>
      - HealthKitExporter.swift or HealthKitExporterViewModel contains `HKHealthStore.isHealthDataAvailable()` guard
      - App launches in iOS Simulator without crash or error log about HealthKit
      - `grep "isHealthDataAvailable" ios/OpenWhoop/HealthKit/HealthKitExporter.swift` → match
    </acceptance_criteria>
  </task>
