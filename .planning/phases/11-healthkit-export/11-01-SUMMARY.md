## Key Files Created/Modified

### key-files.modified
- `ios/project.yml` — added `entitlements.path/properties` block with `com.apple.developer.healthkit: true` and `entitlements.properties.com.apple.developer.healthkit.access: []`; added `NSHealthShareUsageDescription` and `NSHealthUpdateUsageDescription` under `info.properties`

### key-files.created
- `ios/OpenWhoop/OpenWhoop.entitlements` — generated by XcodeGen with `com.apple.developer.healthkit: true` and `com.apple.developer.healthkit.access: []`
- `ios/OpenWhoop/Info.plist` — regenerated with both NSHealth plist keys

## Deviations

- The `capabilities: com.apple.developer.healthkit: {}` syntax (used in the plan) was replaced by `entitlements.properties` syntax, which is the correct XcodeGen 2.45 approach that actually writes the `.entitlements` file. The `capabilities:` key alone generates a `CODE_SIGN_ENTITLEMENTS` reference but writes an empty entitlements file — using `entitlements.properties` writes the correct content.
