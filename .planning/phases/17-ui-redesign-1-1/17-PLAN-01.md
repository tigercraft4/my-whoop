# Plan 17-01 — UI-01: Extract IPA Colors & Update DesignTokens

## Objective

Extract the real WHOOP hex color values from the IPA's `Assets.car`, update `WH.Color.*` in `DesignTokens.swift` with verified values, and commit. This is the mandatory gate (D-01, D-02, D-03): nothing in UI-02/UI-03 may start until this commit exists.

## Context

The current `DesignTokens.swift` has approximate colors (e.g. `#16EC06` for recoveryGreen). The WHOOP 5.37.0 IPA contains the real design system colors in `Assets.car`. The `assetutil` tool (shipped with Xcode) can dump color values as JSON.

IPA path: `re/ghidra/APPS/com.whoop.iphone_5.37.0_und3fined.ipa`
Target file: `ios/OpenWhoop/Design/DesignTokens.swift`
Token names from `docs/specs/v4-ui-map.md`: `recoveryGreen`, `sleepPerformanceDarkBlue`, `lowStrainBlue`, `mediumStrainBlue`, `highStrainBlue`, `backgroundBlack`, `veryDarkBlue`, etc.

## Tasks

### Task 1.1 — Extract Assets.car colors via assetutil

<read_first>
- `docs/specs/v4-ui-map.md` (complete token name list — §Design System Colors)
- `re/ghidra/APPS/com.whoop.iphone_5.37.0_und3fined.ipa` (source IPA)
- `ios/OpenWhoop/Design/DesignTokens.swift` (current tokens to update)
</read_first>

<action>
Run the extraction pipeline:

```
mkdir -p /tmp/whoop_assets
unzip -q "re/ghidra/APPS/com.whoop.iphone_5.37.0_und3fined.ipa" -d /tmp/whoop_assets
assetutil -I /tmp/whoop_assets/Payload/Whoop.app/Assets.car 2>/dev/null
```

Pipe output through `grep` to collect entries where `"Name"` matches the semantic tokens listed in `v4-ui-map.md` §Semantic Color Tokens. For each token found:
- Extract `"Red"`, `"Green"`, `"Blue"` (0.0–1.0 float) or direct hex values from the assetutil JSON
- Convert to 6-char uppercase hex: `R=int(r*255), G=int(g*255), B=int(b*255)` → `#RRGGBB`
- Record: token name → hex, dark-mode variant if present

Priority tokens (update these first, fall back to current value if not found):
1. `backgroundBlack` → maps to `WH.Color.background`
2. `veryDarkBlue` → maps to `WH.Color.surface`
3. `recoveryGreen` → maps to `WH.Color.recoveryGreen`
4. `sleepPerformanceDarkBlue` → maps to `WH.Color.sleepPurple` (rename semantic meaning)
5. `lowStrainBlue` → maps to `WH.Color.strainBlue` (low zone)
6. `mediumStrainBlue` → new token `WH.Color.strainBlueMedium`
7. `highStrainBlue` → new token `WH.Color.strainBlueHigh`
8. `sleepNeedGreen` → new token `WH.Color.sleepNeedGreen`
9. `teal` → maps to `WH.Color.teal`
10. `grey50` → maps to `WH.Color.textSecondary`

If assetutil output is empty or Assets.car not extractable, try:
```
unzip -p "re/ghidra/APPS/com.whoop.iphone_5.37.0_und3fined.ipa" "Payload/Whoop.app/Assets.car" > /tmp/assets.car
assetutil -I /tmp/assets.car 2>/dev/null | head -500
```
</action>

<acceptance_criteria>
- Shell command `assetutil -I /tmp/whoop_assets/Payload/Whoop.app/Assets.car 2>/dev/null | grep -c '"Name"'` returns a count > 0 (colors extracted)
- At least 5 semantic token names from `v4-ui-map.md` appear in the assetutil output (grep for `recoveryGreen\|strainBlue\|backgroundBlack\|veryDarkBlue\|sleepPerformance`)
- Extracted hex values are recorded (in a temporary notes comment in DesignTokens.swift or a local scratch) before editing
</acceptance_criteria>

### Task 1.2 — Update WH.Color.* in DesignTokens.swift

<read_first>
- `ios/OpenWhoop/Design/DesignTokens.swift` (full file — current token values and Color(hex:) extension)
- Extraction output from Task 1.1 (hex values per token name)
- `docs/specs/v4-ui-map.md` §Semantic Color Tokens (semantic mapping table)
</read_first>

<action>
Edit `ios/OpenWhoop/Design/DesignTokens.swift` — `WH.Color` enum:

1. **Update existing tokens** with extracted hex values:
   - `background`: update with `backgroundBlack` hex (currently `#0B0B0F`)
   - `surface`: update with `veryDarkBlue` hex (currently `#16171C`)
   - `surface2`: keep or update with second surface-level color if found
   - `recoveryGreen`: update with `recoveryGreen` extracted hex (currently `#16EC06`)
   - `textSecondary`: update with `grey50` hex (currently `#8A8F98`)
   - `strainBlue`: update with `lowStrainBlue` hex (currently `#0093E7`)
   - `sleepPurple`: update with `sleepPerformanceDarkBlue` hex and rename comment to reflect semantic use

2. **Add new tokens** (after existing ones, before `ringTrack`):
   - `static let strainBlueMedium = SwiftUI.Color(hex: "{mediumStrainBlue hex}")` — medium zone (10–17)
   - `static let strainBlueHigh = SwiftUI.Color(hex: "{highStrainBlue hex}")` — high zone (18–21)
   - `static let sleepNeedGreen = SwiftUI.Color(hex: "{sleepNeedGreen hex}")` — sleep need indicator
   - `static let recoveryDarkerGreen = SwiftUI.Color(hex: "{recoveryDarkerGreen hex}")` — recovery gradient end (if found)

3. **Add section comment** above new tokens:
   ```swift
   // MARK: - Verified from WHOOP 5.37.0 Assets.car (Phase 17 UI-01)
   ```

4. If a token name is NOT found in assetutil output: keep the current approximate value and add inline comment `// approximate — not found in Assets.car`.

5. **Do not change**: `ringTrack`, `stageDeep/Light/Rem/Wake`, `recoveryColor()` function signature, `Color(hex:)` extension.
</action>

<acceptance_criteria>
- `ios/OpenWhoop/Design/DesignTokens.swift` compiles: `cd ios && xcodegen generate && xcodebuild build -scheme OpenWhoop -destination 'platform=iOS Simulator,name=iPhone 17 Pro' 2>&1 | grep -E "error:|BUILD SUCCEEDED"` → BUILD SUCCEEDED
- File contains `strainBlueMedium` and `strainBlueHigh` declarations
- File contains `sleepNeedGreen` declaration
- All `WH.Color.*(hex:)` values are 6-char uppercase hex (no lowercase, no 8-char alpha)
- `grep -c "// approximate" ios/OpenWhoop/Design/DesignTokens.swift` returns count of tokens not found in Assets.car (0 is ideal; ≤5 acceptable)
</acceptance_criteria>
