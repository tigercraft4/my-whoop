# Plan 17-03 — UI-02/03: Home/Recovery Screen Redesign + Snapshot

## Objective

Redesign `RecoveryCard.swift` and `TodayView.swift` to match the official WHOOP Home/Overview screen hierarchy from `v4-ui-map.md`. Add `RecoveryCardSnapshotTests.swift` in `OpenWhoopTests`. Take a simulator screenshot and compare visually to the Ghidra map.

## Context

From `docs/specs/v4-ui-map.md` §Home/Overview, the screen shows:
- 4 circular ring gauges (Recovery, Sleep, Strain, Performance) in a `ScoreBarView`
- A horizontal metric row (RECOVERY | HRV | STRAIN | CALORIES)
- Background: `backgroundBlack` token; card surface: `veryDarkBlue` token

Current `RecoveryCard.swift` uses `Color.black` for background instead of `WH.Color.surface`. Uses a single recovery ring, not 4 gauges. The TodayView already has the right structure but needs token updates.

Snapshot test: uses `SnapshotTesting` 1.17.6, dark mode only, `iPhone17Pro` layout.

Create `ios/OpenWhoopTests/SnapshotTests/RecoveryCardSnapshotTests.swift`:

```swift
import XCTest
import SnapshotTesting
import SwiftUI
@testable import OpenWhoop

final class RecoveryCardSnapshotTests: XCTestCase {

    func testRecoveryCard_green() {
        let daily = DailyMetric(day: "2026-06-01", totalSleepMin: 435,
                                 efficiency: 0.84, deepMin: 90, remMin: 72, lightMin: 180,
                                 disturbances: 3, restingHr: 56, avgHrv: 52,
                                 recovery: 0.73, strain: 14.2, exerciseCount: nil,
                                 respRateBpm: 15.8, sleepPerformance: 84.0)
        let view = RecoveryCard(daily: daily)
            .preferredColorScheme(.dark)
            .frame(width: 390)

        assertSnapshot(of: view, as: .image(on: .iPhone13Pro))
    }

    func testRecoveryCard_empty() {
        let view = RecoveryCard(daily: nil)
            .preferredColorScheme(.dark)
            .frame(width: 390)

        assertSnapshot(of: view, as: .image(on: .iPhone13Pro))
    }
}
```

Notes:
- Use `.image(on: .iPhone13Pro)` (available in SnapshotTesting 1.17.6; `iPhone17Pro` may not be defined — use the closest available device)
- Import pattern must match existing test file imports in `OpenWhoopTests`
- The `SnapshotTests/` subdirectory must be created
- `DailyMetric` constructor: match exact parameter order from `SleepCard.swift` preview to avoid compile errors
</action>
