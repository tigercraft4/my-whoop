### Task 5.2 — Measure spacing and radii via snapshot_ui

<action>
For each screen (Today/Home, Sleep, Strain, Trends):

1. Navigate to the screen in simulator (via tap if needed)
2. `mcp__xcodebuildmcp__snapshot_ui` — capture view hierarchy with coordinates
3. From the view hierarchy, extract:
   - Card corner radius (from `RoundedRectangle` layer bounds vs visual edges)
   - Card internal padding (spacing between card edge and first text element)
   - Ring gauge diameter and lineWidth
   - Stats row column spacing

Compare each measured value against current `WH.Spacing.*` and `WH.Radius.*` tokens:
- `WH.Spacing.md` = 16pt → expected card padding from WHOOP app
- `WH.Spacing.lg` = 24pt → expected recovery card padding
- `WH.Radius.card` = 18pt → expected card corner radius
- `WH.Radius.chip` = 10pt → expected pill/chip radius

If a measured value differs significantly (>3pt) from current token: update `DesignTokens.swift` with the measured value and add comment `// measured via snapshot_ui Phase 17`.
</action>

<acceptance_criteria>
- `mcp__xcodebuildmcp__snapshot_ui` returns valid JSON for each screen
- Any spacing/radius adjustments are applied to `DesignTokens.swift`
- If no adjustments needed: add a comment to DesignTokens.swift `// WH.Spacing.* and WH.Radius.* confirmed via snapshot_ui Phase 17`
- `cd ios && xcodebuild build -scheme OpenWhoop -destination 'platform=iOS Simulator,name=iPhone 17 Pro'` → BUILD SUCCEEDED after any token adjustments
</acceptance_criteria>
