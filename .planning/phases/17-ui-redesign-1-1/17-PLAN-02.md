# Plan 17-02 ‚Äî UI-03 Setup: swift-snapshot-testing SPM Dependency

## Objective

Add `swift-snapshot-testing` 1.17.6 as an SPM package to `ios/project.yml` (under the `OpenWhoopTests` target) and verify the Xcode project regenerates and builds cleanly. No snapshot test code is written in this plan ‚Äî only the dependency infrastructure.

## Context

- `ios/project.yml` currently has two local packages: `WhoopProtocol` and `WhoopStore`
- `OpenWhoopTests` target exists and already has unit tests (MigrationTests, MetricsCacheTests)
- `swift-snapshot-testing` from pointfreeco at `https://github.com/pointfreeco/swift-snapshot-testing`, exact version `1.17.6`
- Decision D-09: add as `exactVersion: 1.17.6` in the `packages:` section, link to `OpenWhoopTests` target only

## Tasks

### Task 2.1 ‚Äî Add SnapshotTesting package to project.yml

<read_first>
- `ios/project.yml` (full file ‚Äî current packages and targets structure)
</read_first>

<action>
Edit `ios/project.yml`:

1. Under the top-level `packages:` key (after existing local packages `WhoopProtocol` and `WhoopStore`), add:
   ```yaml
   SnapshotTesting:
     url: https://github.com/pointfreeco/swift-snapshot-testing
     exactVersion: 1.17.6
   ```

2. Under `targets.OpenWhoopTests.dependencies:`, add after the existing package dependencies:
   ```yaml
   - package: SnapshotTesting
   ```

3. Do NOT add `SnapshotTesting` as a dependency of the main `OpenWhoop` target ‚Äî tests only.
</action>

<acceptance_criteria>
- `ios/project.yml` contains `SnapshotTesting:` key under `packages:`

- `SnapshotTesting` does NOT appear under `targets.OpenWhoop.dependencies`
</acceptance_criteria>

### Task 2.2 ‚Äî Regenerate and build

<read_first>
- `ios/project.yml` (updated file)
</read_first>

<action>
Run the regeneration and build gate:

```bash
cd ios
xcodegen generate
xcodebuild -resolvePackageDependencies -scheme OpenWhoop
xcodebuild build -scheme OpenWhoop -destination 'platform=iOS Simulator,name=iPhone 17 Pro'
```

If `xcodebuild -resolvePackageDependencies` fails due to network:
- Confirm the URL is correct: `https://github.com/pointfreeco/swift-snapshot-testing`
- Ensure version `1.17.6` exists (it was released 2024-02-08, well before cutoff)
- Try `xcodebuild build` directly; SPM may resolve inline

If build fails with "No such module 'SnapshotTesting'" in test files that don't yet import it ‚Äî that is expected (no snapshot imports exist yet). The main app target must build cleanly.
</action>

<acceptance_criteria>
- `xcodegen generate` exits 0 with no errors
- `xcodebuild build -scheme OpenWhoop -destination 'platform=iOS Simulator,name=iPhone 17 Pro'` exits 0 (BUILD SUCCEEDED)
