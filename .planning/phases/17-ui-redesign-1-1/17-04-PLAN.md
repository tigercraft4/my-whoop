# Plan 17-04 — UI-02/03: Sleep + Strain + Trends Screens Redesign + Snapshots

## Objective

Apply WHOOP 1:1 redesign to Sleep, Strain, and Trends screens. Add snapshot tests for SleepCard and StrainCard. Stub Coaching, Health, Profile tabs. Validate each screen in the simulator. All WH.* tokens — no hardcoded values.

## Context

From `docs/specs/v4-ui-map.md`:
- **Sleep**: `sleepPerformanceDarkBlue` ring fill, `highSleepBlue/mediumSleepBlue/lowSleepBlue` stage colors, sleep need indicator in `sleepNeedGreen`
- **Strain**: zone-aware ring (lowStrainBlue / mediumStrainBlue / highStrainBlue), Training State badge
- **Trends**: TrendChartCard uses `strainAccent`/`recoveryGreen` per metric type

All cards must replace `Color.black` with `WH.Color.surface`.

2. **Zone-aware ring color**: replace `ringColor` computed property to return zone-aware color:
   ```
   nil → WH.Color.ringTrack
   strainValue < 10 → WH.Color.strainBlue (lowStrainBlue)
   strainValue < 18 → WH.Color.strainBlueMedium
   else → WH.Color.strainBlueHigh
   ```
   Also update `trainingStateBadgeColor` to use consistent zone mapping:
   - "RESTORATIVE" → `WH.Color.strainBlue` (low zone, blue)
   - "OPTIMAL" → `WH.Color.recoveryGreen`
   - "OVERREACHING" → `WH.Color.recoveryRed`

3. **Stats row**: add a stats row below the ring with RECOVERY | CALORIES as secondary metrics (matching WHOOP screen). Format:
   - RECOVERY: `daily?.recovery.map { "\(Int(($0*100).rounded()))%" } ?? "—"`
   - CALORIES: `daily?.totalCaloriesKcal.map { "\(Int($0)) kcal" } ?? "—"`
   Use same `statColumn()` pattern from RecoveryCard.

4. Ensure no `Color.black` or hardcoded hex remains.
</action>
