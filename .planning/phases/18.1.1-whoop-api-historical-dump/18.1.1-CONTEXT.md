<decisions>
## Implementation Decisions

### O que já existe (NÃO reimplementar)

- **D-01:** `server/ingest/app/whoop_api/client.py` — cliente OAuth2 completo (WhoopClient, build_auth_url, exchange_code). Não tocar.
- **D-02:** `server/ingest/app/whoop_api/__main__.py` — CLI com `auth` e `pull` commands. Não tocar.
- **D-03:** `server/ingest/app/whoop_api/models.py` — GroundTruthDay + GroundTruthWorkout com todos os campos: recovery_score, hrv_rmssd_milli, spo2_percentage, skin_temp_celsius, sleep_performance_pct, day_strain, etc.
- **D-04:** `server/ingest/app/store.py` — `upsert_daily_metrics()` já existe. Pode ser adaptado para receber GroundTruthDay.

### Setup (manual — fazer pelo utilizador)

- **D-05:** Registar app em https://developer-dashboard.whoop.com/apps/create com scopes: read:recovery, read:sleep, read:cycles, read:workout, read:body_measurement, read:profile, offline. Redirect URI: https://localhost/callback
- **D-06:** Obter WHOOP_CLIENT_ID e WHOOP_CLIENT_SECRET do dashboard
- **D-07:** Correr `python -m app.whoop_api auth` para obter WHOOP_REFRESH_TOKEN (processo interactivo)
- **D-08:** Guardar tokens em .env (NUNCA commitar)

### Pull histórico completo

- **D-09:** Data de início: usar a data do primeiro uso do WHOOP (approximar: 2023-01-01 ou pesquisar na app). Data de fim: hoje (2026-06-01).
- **D-10:** Comando: `python -m app.whoop_api pull --start 2023-01-01 --end 2026-06-01 --out /tmp/whoop_history.json`
- **D-11:** Output: JSON com `days[]` (GroundTruthDay por ciclo) e `workouts[]` (GroundTruthWorkout)

### Import para DB local

- **D-12:** Criar `server/ingest/app/whoop_api/import_history.py` — script que lê o JSON e faz upsert de cada GroundTruthDay em `dailyMetric` via `store.upsert_daily_metrics()`. Mapping:
  - `recovery_score` → `recovery` (dividir por 100 para 0.0–1.0)
  - `hrv_rmssd_milli` → `avgHrv`
  - `resting_hr` → `restingHr`
  - `spo2_percentage` → `spo2Pct`
  - `skin_temp_celsius` → `skinTempDevC` (desvio do baseline — a API devolve desvio, não absoluto)
  - `respiratory_rate` → `respRateBpm`
  - `sleep_performance_pct` → `sleepPerformance`
  - `total_sleep_min` (property) → `totalSleepMin`
  - `day_strain` → `strain`
  - `deep_sleep_min`, `rem_sleep_min`, `light_sleep_min` → `deepMin`, `remMin`, `lightMin`
- **D-13:** Usar `device_id` do .env (WHOOP_DEVICE_ID) para o campo deviceId na DB

### O que não fazer nesta fase

- **D-14:** Não implementar sync contínuo agora — só o dump histórico one-shot. Sync contínuo é Fase 999.3.
- **D-15:** Não modificar a app iOS nesta fase — apenas o servidor.

</decisions>
