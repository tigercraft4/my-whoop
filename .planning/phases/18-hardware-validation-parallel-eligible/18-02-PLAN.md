SETUP (fazer UMA VEZ para os 3 PROTOs):

2. Ter disponivel: oximetro de pulso (Fingertip Pulse Oximeter) e termometro de contacto/IR
3. Conectar WHOOP 5.0 via app OpenWhoop — confirmar que esta a receber dados
4. Activar TOGGLE_IMU_MODE (comando 0x6A = 106) via o command sender da app. Se nao existir botao dedicado, abrir BLE command sender e enviar `toggleIMUMode` com payload [0x01]

6. Capturar durante 5 minutos

PROTO-11 — SpO2:
7. Colocar o oximetro no mesmo dedo/mao que o WHOOP 5.0 esta no pulso
8. Registar leituras do oximetro a cada 30 segundos durante 5 min (minimo 5 leituras)
9. Nota dos valores vistos no ecra da app (spo2_red, spo2_ir) ou nos logs PacketLogger
10. Verificar: |WHOOP_decoded - oximetro_leitura| <= 2% em 5 medicoes consecutivas => VERIFIED

PROTO-12 — Skin Temperature:
11. Usar termometro de contacto/IR para medir temperatura da pele no pulso durante a sessao
12. Comparar com o valor decoded de skin_temp (converter de raw ADC se necessario)
13. Verificar: desvio plausivel (0.5C de margem para condicoes ambiente) => VERIFIED

PROTO-13 — Respiration Rate:
14. Contar respiracoes manualmente durante 1 minuto (ou usar oximetro com funcao resp rate)
15. Comparar com o valor decoded de resp_rate (em rpm)
16. Verificar: 12-20 rpm com desvio <= 2 rpm => VERIFIED
17. SE nenhum campo de respiration for observado no wire (frames type-47), confirmar como CLOUD_DERIVED

GUARDAR:
- Exportar capture PacketLogger como .pklg (re/capture/evidence/ — GITIGNORED, raw biometrics)
- Fotografar/registar leituras dos instrumentos ground truth
