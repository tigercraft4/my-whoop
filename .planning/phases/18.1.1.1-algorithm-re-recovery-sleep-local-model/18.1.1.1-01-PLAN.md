# Plan 18.1.1.1-01 — Calibração Recovery: regressão linear HRV+RHR → WHOOP score

## Objective

O iOS usa `score = 66.0 * (hrv/baseline) + 33.0` para o Recovery score. Com dados históricos reais da WHOOP (importados pela Fase 18.1.1), é possível calibrar os coeficientes por regressão linear simples. Este plano:
1. Cria script Python que lê o GRDB, faz regressão, exporta coeficientes JSON
2. Actualiza `LocalMetricsComputer.swift` para usar os coeficientes calibrados

## Context

Os dados históricos estão no GRDB iOS (SQLite). A tabela `dailyMetric` tem:
- `avgHrv` (ms) — HRV RMSSD overnight
- `restingHr` (bpm) — resting HR
- `recovery` (0.0–1.0) — ground truth WHOOP (importado pelo `import_csv_to_grdb.py`)

Algoritmo actual iOS: `ratio = hrv / hrv_28day_mean`, `score = 66.0 * ratio + 33.0`
Objetivo: calibrar `a` e `b` em `score = a * ratio + b` usando sklearn.LinearRegression.

## Tasks

### Task 1.1 — Script `calibrate_recovery.py`

<read_first>
- `server/ingest/app/whoop_api/import_csv_to_grdb.py` — schema da tabela dailyMetric
- `ios/OpenWhoop/Metrics/LocalMetricsComputer.swift` linhas 266–278 — fórmula actual
- `server/ingest/app/analysis/recovery.py` — modelo servidor (referência)
</read_first>

Criar `server/ingest/app/whoop_api/calibrate_recovery.py`:

```python
"""
calibrate_recovery.py — Calibrar coeficientes de Recovery usando dados históricos WHOOP.

Usage:
    python calibrate_recovery.py --db /path/to/whoop.sqlite --out recovery_coefficients.json

Lê dailyMetric, computa hrv_ratio (28-day rolling baseline), faz LinearRegression,
exporta coeficientes para JSON.
"""
```

**Lógica principal:**
1. `sqlite3.connect(db_path)` → SELECT day, avgHrv, restingHr, recovery FROM dailyMetric WHERE avgHrv IS NOT NULL AND recovery IS NOT NULL ORDER BY day
2. Computar 28-day rolling baseline para HRV: `hrv_baseline[i] = mean(hrv[max(0,i-28):i])`
3. Feature: `hrv_ratio = avgHrv / hrv_baseline` (excluir rows onde baseline não disponível, i.e., < 3 dias)
4. Target: `recovery * 100` (0–100)
5. `from sklearn.linear_model import LinearRegression; model = LinearRegression(); model.fit(X, y)`
6. Onde X = `[[hrv_ratio]]` (modelo 1D simples, matches iOS formula)
7. Se `rhr` disponível: testar também X = `[[hrv_ratio, rhr_ratio]]` e reportar R² de ambos
8. Escolher o modelo mais simples com R² aceitável (≥ 0.30)
9. Reportar: R², MAE, comparação com fórmula actual
10. Output JSON: `{"hrv_slope": float, "hrv_intercept": float, "rhr_slope": float|null, "r2": float, "mae": float, "n_samples": int}`
11. `--out` flag para salvar JSON

**Self-check:** `python calibrate_recovery.py --db whoop.sqlite --out /tmp/coef.json` → JSON válido com R² > 0.20, n_samples > 14.

### Task 1.3 — Actualizar `LocalMetricsComputer.swift`

<read_first>
- `ios/OpenWhoop/Metrics/LocalMetricsComputer.swift` linhas 266–278
</read_first>

Actualizar `computeRecovery` para carregar e usar os coeficientes do JSON:

1. Adicionar método privado `loadRecoveryCoefficients() -> (slope: Doercept: Double)`:
   ```swift
   private func loadRecoveryCoefficients() -> (slope: Double, intercept: Double) {
       guard let url = Bundle.main.url(forResource: "recovery_coefficients", withExtension: "json"),
             let data = try? Data(contentsOf: url),
             let obj = try? JSONSerialization.jsonObject(with: data) as? [String: Double]
       else { return (slope: 66.0, intercept: 33.0) }  // fallback aos valores actuais
       return (slope: obj["hrv_slope"] ?? 66.0, intercept: obj["hrv_intercept"] ?? 33.0)
   }
   ```
2. Actualizar `computeRecovery` — substituir `ratio * 66.0 + 33.0` por coeficientes carregados:
   ```swift
   let (slope, intercept) = loadRecoveryCoefficients()
   let score = min(100.0, max(0.0, slope * ratio + intercept))
   ```
3. O clamp muda de `(33, 100)` para `(0, 100)` — o intercept calibrado é o mínimo natural.
