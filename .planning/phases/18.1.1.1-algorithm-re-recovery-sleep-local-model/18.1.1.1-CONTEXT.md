<decisions>
## Implementation Decisions

### Abordagem de regressão (confirmada com utilizador)

- **D-01:** Regressão linear simples: `sklearn.LinearRegression` com features HRV RMSSD + RHR → Recovery score 0–100
- **D-02:** Feature principal: `hrv_ratio = avgHrv / hrv_28day_baseline` (normalização baseline, coerente com algoritmo actual do iOS)
- **D-03:** Feature secundária: `rhr_ratio = restingHr / rhr_28day_baseline` (incluir se melhora R²)
- **D-04:** Target: `recovery * 100` (0–100, ground truth WHOOP da coluna `recovery` do GRDB)
- **D-05:** Exportar coeficientes como JSON: `recovery_coefficients.json` → bundled no app iOS

### Algoritmo actual (a calibrar)

- **D-06:** iOS `computeRecovery`: `ratio = hrv / baseline`, `score = ratio * 66.0 + 33.0` → substituir `66.0` e `33.0` pelos coeficientes ajustados
- **D-07:** Servidor `recovery.py`: z-score + logístico com W_HRV=0.60, W_RHR=0.20 — NÃO modificar (é o modelo do servidor, que já é mais sofisticado)

### Scope

- **D-08:** Recovery APENAS nesta fase. Sleep Performance é validação de comparação (não alterar fórmula iOS — já usa pesos razoáveis).
- **D-09:** O script Python corre localmente com o GRDB SQLite exportado do iOS Simulator.
- **D-10:** Mínimo de 14 dias com dados não-nulos para a regressão ser válida. Reportar R² e MAE.

</decisions>
