## Summary

Calibração completa do algoritmo de Recovery com base em 30 dias de dados históricos reais da WHOOP. A fórmula hardcoded (`66 × ratio + 33`) foi substituída por coeficientes ajustados por regressão linear (R²=0.49, MAE=12.0 pts), carregados dinamicamente de `recovery_coefficients.json`.

## What was built

- **`server/ingest/app/whoop_api/calibrate_recovery.py`** (174 linhas):
  - Lê dailyMetric do GRDB iOS (SQLite)
  - Computa rolling 28-day baseline para HRV
  - Feature: `hrv_ratio = avgHrv / hrv_baseline`
  - Fit OLS: `score = hrv_slope × hrv_ratio + hrv_intercept`
  - Reporta R², MAE, comparação com fórmula antiga
  - Exporta JSON com coeficientes calibrados

- **`ios/OpenWhoop/Resources/recovery_coefficients.json`**:
  ```json
  {
    "hrv_slope": 54.4833,
    "hrv_intercept": -0.2683,
    "r2": 0.4877,
    "mae": 11.98,
    "n_samples": 30,
    "baseline_window_days": 28
  }
  ```

- **`ios/OpenWhoop/Metrics/LocalMetricsComputer.swift`** — `computeRecovery()` actualizado:
  - Carrega coeficientes do JSON via `Bundle.main.url(forResource:)`
  - Fallback seguro: (slope: 54.48, intercept: -0.27) se JSON não disponível
  - Clamp alterado de `(33, 100)` para `(0, 100)` — mais correcto com intercept ~0
  - `static let recoveryCoefficients` — lazy loaded once at startup

## Results

| Métrica | Fórmula antiga | Calibrada |
|---------|---------------|-----------|
| R²      | -2.93         | +0.49     |
| MAE     | 46.4 pts      | 12.0 pts  |
| Melhoria MAE | —        | 74%       |

Interpretação: ratio=1.0 (HRV na baseline) → Recovery≈54% (coerente com média WHOOP 58%).

## Deviations

- `rhr_ratio` não foi incluído como feature (contribuição marginal com apenas 30 amostras). Pode ser adicionado quando houver mais histórico (>90 dias).
- Intercept ≈ 0 → a fórmula simplifica para `score ≈ 54.48 × ratio`, mais interpretável.
