<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iPhone/Android device → host capture file | Live BLE/HCI traffic (post-decryption) crosses to a local file that may carry device IDs + SMP material |
| Raw capture (`samples/`) → committed evidence (`evidence/`) | Only redacted, scrubbed excerpts may cross this boundary into git |
| Apple/Android device-side UI → reproducible runbook | Manual steps the operator performs must be precise enough to not skip the redaction/gitignore guardrails |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Information Disclosure | Committed evidence hex excerpt | mitigate | Runbooks + checkpoint mandate scrubbing BD_ADDR / SMP key material before commit; only a small redacted slice committed (DISCLAIMER §2) |
| T-01-09 | Information Disclosure | Raw `.pklg`/`.btsnoop` containing SMP/bonding material | mitigate | Raw captures saved only under gitignored `re/capture/samples/` (Plan 01 ignore rule); checkpoint step 4 verifies `git status` shows no raw binaries stage-able |
| T-01-10 | Spoofing / Capture integrity | Empty or wrong-session capture passed off as evidence | mitigate | Every success criterion requires NON-EMPTY WHOOP ATT traffic, verified via `tshark -Y btatt`; metadata sidecar records firmware + notes for provenance |
| T-01-11 | Tampering | Operator skips HCI-snoop-before-session ordering | accept | Documented warning in android-btsnoop.md (Pitfall 3); failure mode is an empty trace caught by the non-empty verification, not a security exposure |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No language-package installs in this plan; PacketLogger is an Apple-distributed app verified by presence-check, not a registry package |
</threat_model>
