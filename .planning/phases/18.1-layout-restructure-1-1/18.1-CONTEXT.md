# Phase 18.1: Layout Restructure 1:1 - Context

**Gathered:** 2026-06-01
**Status:** Ready for planning
**Reference:** Screenshots of WHOOP 5.37.0 iOS app (pt-PT, 23:08/23:16)

<domain>
## Phase Boundary

Reestruturar o layout completo da app OpenWhoop para corresponder visualmente à app WHOOP 5.37.0:
1. Tab bar: Início | Saúde | Comunidade | Mais + botão W
2. Home (Início): header com data nav + 3 rings em linha + secções WHOOP
3. Saúde tab: métricas de saúde locais (HRV, RHR, SpO₂, tendências sono)
4. Comunidade + Mais: stubs com placeholder

Sem mudanças de dados ou algoritmos — apenas estrutura de layout e navegação.

</domain>

<decisions>
## Implementation Decisions

### Tab Bar — RootTabView.swift

- **D-01:** Substituir 5 tabs actuais (Today/Sleep/Strain/Trends/Device) por:
  - Tab 1: **Início** — ícone `house` → HomeView (novo nome para TodayView reestruturado)
  - Tab 2: **Saúde** — ícone `heart.text.clipboard` → HealthView (novo)
  - Tab 3: **Comunidade** — ícone `person.3` → CommunityView (stub)
  - Tab 4: **Mais** — ícone `line.3.horizontal` → MoreView (stub)
- **D-02:** Botão W (dispositivo) — manter o LiveView mas acessível via botão overlay no canto inferior direito (circular, cor `WH.Color.sleepPurple`), não como tab. Replicar o botão "W" da WHOOP (ícone `W` ou WHOOP icon).
- **D-03:** Sleep, Strain, Trends — acessíveis via tap nos rings do ecrã Início (detail push) e possivelmente como sub-secções do Saúde tab. Não são tabs standalone.

### Home Screen (Início) — HomeView.swift

- **D-04:** Header customizado no topo:
  - Left: ícone perfil (circle pessoa) + ícone flame + valor de strain do dia
  - Center: `< HOJE >` — navegação de data (String do dia, setas para trás/frente)
  - Right: `--% + ícone dispositivo` — recovery% (ou "--" se sem dados)
- **D-05:** 3 rings em linha abaixo do header:
  - Ring 1: **SONO** — valor sleepPerformance% ou "--", cor `WH.Color.sleepPurple`
  - Ring 2: **RECUPERAÇÃO** — valor recovery% ou "--", cor recovery-band (green/yellow/red)
  - Ring 3: **ESFORÇO** — valor strain (0–21) ou "--", cor `WH.Color.strainBlue`
  - Cada ring tem label abaixo + seta ">" indicando que é tappável
  - Tap em cada ring → push para SleepView / RecoveryDetailView / StrainView
- **D-06:** Rings são pequenos (~90–100pt diameter) — não o hero grande actual. Usar `ZoneRingView` existente com tamanho reduzido.
- **D-07:** Secções abaixo dos rings (scroll vertical):
  1. **Dados em falta** card — mostrar quando não há dados recentes (>24h sem sync). Texto: "O WHOOP não recebeu dados nas últimas 'N' horas." Dark grey card.
  2. **Monitor de Saúde + Monitor de Stress** — 2 cards lado a lado. Tappáveis (→ HealthView). Mostrar "--" quando sem dados.
  3. **O meu dia** — header de secção com botão [+] (futura funcionalidade — botão + inactivo por agora)
  4. **Atividades de Hoje** card — botões "ADICIONAR ATIVIDADE" + "INICIAR ATIVIDADE" (stub, inactivos)
  5. **Sono desta noite** card — mostrar sleepNeededMin como hora de dormir recomendada + smart alarm existente

### Saúde Tab — HealthView.swift (novo)

- **D-08:** WHOOP Age e Healthspan são cloud-only — não replicáveis. Em vez disso, HealthView mostra as métricas de saúde que temos:
  - Secção HRV: gráfico de tendência dos últimos 7 dias (usar `MetricChart` existente)
  - Secção RHR: gráfico de tendência 7 dias
  - Secção SpO₂: último valor ou "--"
  - Secção Qualidade de Sono: sleepPerformance 7 dias
  - Header: "SAÚDE" (navigation title, centrado)
- **D-09:** Sem tentar replicar WHOOP Age (30,3) ou Advanced Labs — são premium cloud features.

### Comunidade + Mais — Stubs

- **D-10:** `CommunityView`: View simples com `Text("Em breve")` centrado + header "COMUNIDADE"
- **D-11:** `MoreView`: Lista de configurações básicas — DeviceId, App version, link para settings. Header "MAIS".

### Navegação de Data

- **D-12:** Header Início mostra "HOJE" por defeito. Setas ← → permitem navegar entre dias (usar `MetricsRepository.dailyMetrics()` para carregar dados de outras datas). Estado: `@State private var selectedDate: Date = .now`.
- **D-13:** Quando selectedDate != hoje, mostrar data formatada ("SEG 31 MAI") em vez de "HOJE".

## Existing Code Insights

### Reusable Assets
- `ZoneRingView(value:maxValue:color:lineWidth:size:centerLabel:)` — já existe; passar `size: 90` para rings pequenos
- `MetricChart` — gráfico de tendência para HealthView HRV/RHR
- `ScreenHeader` — adaptar ou substituir pelo novo header com navegação de data
- `AlarmView` + `@AppStorage(AlarmKeys.*)` — reutilizar para card "Sono desta noite"
- `MetricsRepository.sleepDetail()`, `.refresh()`, `.today` — fontes de dados existentes

### Established Patterns
- `@EnvironmentObject private var metrics: MetricsRepository` — injectar em HomeView e HealthView
- Todos os valores mostram "—" quando nil — manter
- `.preferredColorScheme(.dark)` no RootTabView — manter
- Tab bar SwiftUI: `TabView(selection: $selectedTab)` + `.tabItem { Label(...) }`

### Integration Points
- RecoveryCard, SleepCard, StrainCard — continuam a existir como detail views acessíveis via tap nos rings
- LiveView — mantém-se mas como overlay button, não tab
- SleepView, StrainView — mantêm-se como push destinations dos rings

</code_context>

## Specific Ideas

- Botão W: `Button { selectedTab = "device" } label: { Circle().fill(WH.Color.sleepPurple).overlay(Text("W")) }` como overlay sobre a tab bar
- Rings em linha: `HStack(spacing: WH.Spacing.lg) { ring1; ring2; ring3 }` centrado horizontalmente
- "HOJE" com setas: `HStack { Button("<") { }; Text(dateLabel); Button(">") { } }` no header
- Card "Dados em falta": mostrar quando `metrics.today == nil && !metrics.isRefreshing && sinceDays > 1`
