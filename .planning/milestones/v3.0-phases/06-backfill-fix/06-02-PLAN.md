- ios/OpenWhoop/BLE/BLEManager.swift (linha 390: BLEManager.logger.notice("BF: session ended reason=...") — padrão de log de fim de sessão)

<action>
1. Adicionar propriedades de tracking de range temporal ao Backfiller (junto às propriedades existentes):
   `private var firstChunkUnix: UInt32? = nil`
   `private var lastChunkUnix: UInt32 = 0`

2. Em finishChunk(unix:trim:endFrame:), após `let frames = chunk`, adicionar:
   ```swift
   if firstChunkUnix == nil { firstChunkUnix = unix }
   lastChunkUnix = unix
   ```

3. No método que processa HISTORY_COMPLETE (metaType == 3 no ingest), após limpar o estado,
   adicionar o log de range:
   ```swift
   if let first = firstChunkUnix, lastChunkUnix > 0 {
       let dayCount = max(1, Int((lastChunkUnix - first) / 86400))
       BLEManager.logger.notice("BF: session ended — range=\(first, privacy: .public)...\(self.lastChunkUnix, privacy: .public) (~\(dayCount, privacy: .public) days)")
   }
   firstChunkUnix = nil
   lastChunkUnix = 0
   ```
   Se Backfiller não tem acesso directo a BLEManager.logger, usar `os.Logger` com o mesmo subsystem/category.
   Verificar qual logger é usado no Backfiller existente e usar o mesmo.

4. Em timeoutFired(), reset das propriedades de range:
   ```swift
   firstChunkUnix = nil
   lastChunkUnix = 0
   ```

Nota: os unix timestamps nos chunks históricos são wall-clock seconds (type-47 é self-timestamped).
Um range de 14+ dias significa `lastChunkUnix - firstChunkUnix >= 14 * 86400`.
</action>
