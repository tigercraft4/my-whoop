<action>
Add the following route to `server/ingest/app/main.py` after the existing `/v1/daily` route:

```
@app.get("/v1/today", dependencies=[Depends(require_auth)])
def get_today(device: str):
    """Most-recent daily_metrics row for the device (ORDER BY day DESC LIMIT 1), or null."""
    with psycopg.connect(cfg.db_dsn) as conn:
        return read.query_today(conn, device)
```

The route returns the dict directly (FastAPI serialises to JSON) or `None` (FastAPI serialises to JSON `null`). No `_parse_date` needed.
</action>

- Route accepts `device: str` query parameter (same style as `/v1/batches`, `/v1/summary`)
- Route does NOT import any new dependencies (psycopg and read already imported)
- `curl -H "Authorization: Bearer <key>" "http://localhost:8000/v1/today?device=my-whoop"` returns either a JSON object with a `day` key or `null` (not a JSON array)
