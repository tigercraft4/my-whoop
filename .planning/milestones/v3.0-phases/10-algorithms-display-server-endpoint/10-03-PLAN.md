<action>
Modify `heroSection` in `TodayView.swift` to include a staleness label below the recovery ring/content.

Change from a plain `HStack { Spacer(); NavigationLink { ... }; Spacer() }` to a `VStack` wrapping the existing `HStack` plus the new staleness label:

```swift
private var heroSection: some View {
    VStack(spacing: WH.Spacing.xs) {
        HStack {
            Spacer()
            NavigationLink(destination: MetricDetailView(kind: .recovery)) {
                // existing ring / pendingRecoveryRing content ‚Äî UNCHANGED
            }
            .buttonStyle(.plain)
            Spacer()
        }
        .padding(.top, WH.Spacing.sm)

        // Staleness label ‚Äî only when lastRefreshedAt > 6h
        if let at = metrics.lastRefreshedAt,
           Date().timeIntervalSince(at) > StalenessPolicy.staleAfterSeconds {
            Text("Updated \(Int(Date().timeIntervalSince(at) / 3600))h ago")
                .font(WH.Font.caption)
                .foregroundStyle(WH.Color.textSecondary)
        }
    }
}
```

Keep the `.padding(.top, WH.Spacing.sm)` on the inner HStack (not the outer VStack) to preserve visual spacing.

The staleness label does NOT replace the `syncFooter` ‚Äî both can be visible simultaneously. The `syncFooter` always shows when `lastRefreshedAt` is set; the staleness label adds an additional signal specifically in the hero area when > 6h stale.
</action>
