# Plan 09-06: TrendsView — IOS-05: SpO₂ e skinTemp

## Objective

Adicionar casos `.spo2` e `.skinTemp` a `MetricKind` e incluí-los em `MetricKind.dailyCases` para que apareçam no carrossel de `TrendChartCard` em `TrendsView`. Dados de `DailyMetric.spo2Pct` e `DailyMetric.skinTempDevC` — mostrar `"—"` se nil (streams HYPOTHESIS não verificados). Este é o IOS-05 deferido de Phase 7.

## Context

`MetricKind` é o enum central que controla charts, cores, extracção de dados e formatação. Para adicionar SpO₂ e skinTemp basta adicionar dois novos casos seguindo o padrão existente. `DailyMetric` já tem `spo2Pct: Double?` e `skinTempDevC: Double?`. `MetricKind.dailyCases` é o array que controla quais métricas aparecem nos TrendChartCards — adicionar `.spo2` e `.skinTemp` aqui.

<action>
1. Em `MetricKind.swift`, adicionar dois novos casos ao enum:
   - `case spo2` — SpO₂ (Blood Oxygen %)
   - `case skinTemp` — Skin Temperature deviation (°C from baseline)

2. Actualizar `static let dailyCases` para incluir os novos casos:
   - `static let dailyCases: [MetricKind] = [.recovery, .hrv, .rhr, .strain, .sleepDuration, .spo2, .skinTemp]`

3. Actualizar cada switch para incluir os novos casos:

   **`var title: String`:**
   - `.spo2`: `return "Blood Oxygen"`
   - `.skinTemp`: `return "Skin Temp"`

   **`var unit: String`:**
   - `.spo2`: `return "%"`
   - `.skinTemp`: `return "°C"`

   **`var color: Color`:**
   - `.spo2`: `return WH.Color.teal` (azul/ciano — distinto das outras métricas)
   - `.skinTemp`: `return SwiftUI.Color(hex: "#FF9F0A")` (laranja — Apple system orange, distinto)
   Nota: consultar `docs/whoop-ui-reference.md` para ver se há cores WHOOP definidas para SpO₂ e skinTemp; usar as cores do reference doc se disponíveis.

   **`var markType: MarkType`:**
   - `.spo2`: `return .line`
   - `.skinTemp`: `return .line`

   **`var fixedYDomain: ClosedRange<Double>?`:**
   - `.spo2`: `return 90...100` (range fisiológico relevante para SpO₂)
   - `.skinTemp`: `return nil` (auto-scale — deviação pode variar)

   **`var hasRecoveryBands: Bool`:**
   - Ambos: `false` (padrão, já coberto pelo `self == .recovery`)

   **`var isStreamBacked: Bool`:**
   - Ambos: `false` (são daily aggregates, não streams)

   **`func format(_ value: Double)`:**
   - `.spo2`: `return String(format: "%.0f%%", value)`
   - `.skinTemp`: `return String(format: "%+.1f °C", value)` (mostrar sinal para desvio de baseline)

   **`func formatShort(_ value: Double)`:**
   - `.spo2`: `return String(format: "%.0f", value)`
   - `.skinTemp`: `return String(format: "%+.1f", value)`

   **`func value(from metric: DailyMetric) -> Double?`:**
   - `.spo2`: `return metric.spo2Pct`
   - `.skinTemp`: `return metric.skinTempDevC`
   
   Nota: se `DailyMetric` não tiver `spo2Pct` ou `skinTempDevC` como propriedades públicas, verificar o WhoopStore model. Se os campos não existirem no modelo, retornar `nil` e deixar comentário explicativo.

4. Garantir que o enum compila sem warnings de "switch not exhaustive".
</action>
