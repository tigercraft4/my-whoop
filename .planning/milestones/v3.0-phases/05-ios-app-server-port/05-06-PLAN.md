## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| strap BLE ↔ iPhone | dados biometricos em transito sobre link bonded encriptado |
| processo da app ↔ WhoopStore | kill abrupto durante escrita; risco de corrupcao/perda |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06-01 | Denial of Service / data loss | kill durante ACK pendente | mitigate | store-then-ack discipline (store -> fsync -> ack); o kill-test prova que dados nao confirmados sao re-enviados pelo strap na reconexao (ingest idempotente) — gate explicito desta fase |
| T-05-06-02 | Repudiation / integridade | backfill com gaps/duplicados | mitigate | safe-trim invariant no Backfiller garante que so se faz trim apos persistencia confirmada; verificado pela ausencia de gaps no kill-test |
| T-05-06-03 | Information Disclosure | dados biometricos no iPhone | accept | App Sandbox iOS + bond encriptado; dados do proprio utilizador; offline-first (servidor opcional) |
| T-05-06-04 | Denial of Service | reconexao em background apos force-quit falha | mitigate | CBCentralManagerOptionRestoreIdentifierKey + willRestoreState validados fisicamente na Task 2 (IOS-08 / ROADMAP SC 4) — sem state restoration, a app nao recolhe dados quando terminada |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Zero packages novos; toolchain Xcode/Swift de sistema (RESEARCH confirma) |
</threat_model>
