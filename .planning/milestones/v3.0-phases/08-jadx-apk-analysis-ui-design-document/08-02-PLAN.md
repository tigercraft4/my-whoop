  <action>
    For each of the 5 tabs, populate the `### Field-to-Model Mapping` subsection with a table in this exact format:

    ```markdown
    | UI Field (Android Label) | iOS Model Property | Requirement |
    |--------------------------|-------------------|-------------|
    | Recovery Score | `DailyMetric.recovery` (Int 0–100) | ALG-01 |
    | HRV | `DailyMetric.hrv` (Double ms) | — |
    | RHR | `DailyMetric.rhr` (Int bpm) | — |
    ```

    Mandatory mappings per tab:

    **Tab 1 (Overview/Today):**
    - Recovery Score → `DailyMetric.recovery` (ALG-01)
    - HRV RMSSD → `DailyMetric.hrv`
    - RHR → `DailyMetric.rhr`
    - Sleep Performance → derived from `CachedSleepSession.efficiency` or `DailyMetric.totalSleepMin`
    - Day Strain → `DailyMetric.strain` (ALG-03)

    **Tab 2 (Sleep):**
    - Sleep Duration → `CachedSleepSession.totalSleepMin` or (`endTs` - `startTs`)
    - Sleep Efficiency → `CachedSleepSession.efficiency`
    - REM minutes → `CachedSleepSession.remMin` (ALG-02)
    - Deep Sleep minutes → `CachedSleepSession.deepMin` (ALG-02)
    - Light Sleep minutes → `CachedSleepSession.lightMin` (ALG-02)
    - Awake → `CachedSleepSession.awakeSec`
    - Disturbances → `CachedSleepSession.disturbances`
    - Staging breakdown → `CachedSleepSession.stagesJSON` (ALG-02)

    **Tab 3 (Strain/Workouts):**
    - Day Strain → `DailyMetric.strain` (ALG-03)
    - HR zones → derived from raw HR samples (no direct iOS model field — mark as `[computed from hrSample stream]`)
    - Workout entries → `Workout` model from `MetricsRepository.workouts()`

    **Tab 4 (Trends):**
    - HR trend → `TrendPoint` series from `MetricsRepository.hrSeries()`
    - HRV trend → `DailyMetric.hrv` series over date range via `MetricsRepository.daily()`
    - SpO2 trend → `DailyMetric.spo2` (PROTO-11 gated)
    - Skin temp trend → `DailyMetric.skinTemp`
    - Respiration trend → `DailyMetric.respirationRate`

    **Tab 5 (Coach/Device/Plus):**
    - Map whatever fields are visible (if paywall — mark as `[premium gated]`)
    - Device status → in-app BLE state, not a DailyMetric field

    For fields with no direct iOS model counterpart, mark the Requirement column as `[computed]` or `[not in local model]` with a note.
