<objective>
Add the HealthKit capability entitlement and the two required plist usage-description keys to ios/project.yml, then regenerate the .xcodeproj via `xcodegen generate`. This is the HK-P1 invariant: entitlements MUST exist in the project before ANY `import HealthKit` appears in Swift code. No Swift code is written in this plan.
</objective>

<threat_model>
- LOW: `NSHealthUpdateUsageDescription` text is user-facing; vague strings may confuse App Review (mitigated: use explicit copy from CONTEXT.md).
- LOW: Running `xcodegen generate` overwrites the .xcodeproj; if additional manual Xcode edits exist they would be lost (mitigated: project uses XcodeGen exclusively, no manual edits).
</threat_model>
